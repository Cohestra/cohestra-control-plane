Flink Control Plane

Open-source deployment management for Apache Flink on Kubernetes. Your cluster, your rules — no vendor lock-in.

Apache 2.0 Any Kubernetes Production Ready

Deploy & Rollback

Controlled rollouts with savepoint management, health gates, and automatic rollback on failure.

Scale & Autoscale

Manual scaling, Flink Operator autoscaler, or plug in your own custom autoscaler via SDK.

Observe & Control

Durable operation history, version tracking, cluster freeze, and a built-in operations console.

FCP vs AWS Managed Service for Apache Flink

AWS MSF

  • Fully managed by AWS — no cluster access
  • Flink versions lag behind OSS releases
  • Locked to AWS, specific regions
  • Limited scaling options, no custom autoscaler
  • Opaque upgrade/rollback process
  • No durable operation history
  • Pricing per KPU — scales with cost at high volume
  • State management is a black box

FCP (Flink Control Plane)

  • Runs on any Kubernetes (EKS, GKE, AKS, on-prem)
  • Flink 2.x on day one — you control the version
  • Open source, Apache 2.0 — no lock-in
  • Custom autoscaler SDK + Flink Operator autoscaler
  • Transparent rollouts with savepoint gates
  • Full operation history via Temporal
  • Pay only for your Kubernetes nodes
  • You own your checkpoints, savepoints, and HA state

Quick Start

1. Install on your Kubernetes cluster

bashhelm repo add fcp https://flink-control-plane.github.io/charts
helm install fcp fcp/fcp \
  --namespace fcp-system --create-namespace \
  --set temporal.enabled=true

2. Register a deployment

bashcurl -X PUT https://fcp.yourcluster:8080/api/v1/deployments/prod/streaming/orders \
  -H 'Content-Type: application/json' \
  -d '{"owner":"platform","serviceAccount":"flink"}'

3. Deploy your Flink job

bashcurl -X POST https://fcp.yourcluster:8080/api/v1/deployments/prod/streaming/orders/deploy \
  -H 'Content-Type: application/json' \
  -H 'Idempotency-Key: deploy-001' \
  -d '{
    "requester": "ci",
    "approved": true,
    "spec": {
      "imageDigest": "your-registry/orders-job@sha256:...",
      "flinkVersion": "2.2",
      "parallelism": 8,
      "maxParallelism": 128,
      "resources": {
        "taskManagerCpu": 2,
        "taskManagerMemoryMiB": 4096,
        "taskManagerCount": 2,
        "slotsPerManager": 4
      },
      "stateCompatibility": {
        "jobGraphCompatible": true,
        "operatorUidsStable": true
      }
    }
  }'

4. Or use the Python SDK

Custom Autoscaler

FCP provides an AutoscalerBase class for building custom autoscalers. Subclass it, implement evaluate(), and deploy as a Lambda, CronJob, or long-running process. 

pythonfrom fcp_sdk import FCPClient, AutoscalerBase, ScaleDecision

class BackpressureAutoscaler(AutoscalerBase):
    def evaluate(self, status: dict) -> ScaleDecision | None:
        health = status["currentVersion"]["healthSummary"]
        current_p = status["currentVersion"]["spec"]["parallelism"]

        if health["backpressureRatio"] > 0.7 and current_p < 64:
            return ScaleDecision(
                target_parallelism=min(current_p * 2, 64),
                reason=f"backpressure={health['backpressureRatio']:.0%}",
            )

        if health["backpressureRatio"] < 0.1 and current_p > 4:
            return ScaleDecision(
                target_parallelism=max(current_p // 2, 4),
                reason="low utilization",
            )

        return None

python — kafka_lag_autoscaler.pyimport math, os
from fcp_sdk import FCPClient, AutoscalerBase, ScaleDecision

class KafkaLagAutoscaler(AutoscalerBase):
    def evaluate(self, status):
        lag = status["currentVersion"]["healthSummary"]["kafkaLag"]
        current = status["currentVersion"]["spec"]["parallelism"]
        lag_per_slot = int(os.environ.get("LAG_PER_SLOT", "50000"))
        max_p = int(os.environ.get("MAX_PARALLELISM", "64"))

        target = min(max(math.ceil(lag / lag_per_slot), 2), max_p)
        if target != current:
            return ScaleDecision(target, reason=f"lag={lag}")
        return None

def handler(event, context):
    client = FCPClient(os.environ["FCP_URL"], token=os.environ.get("FCP_TOKEN"))
    scaler = KafkaLagAutoscaler(
        client, os.environ["FCP_ENV"],
        os.environ["FCP_NAMESPACE"], os.environ["FCP_NAME"],
    )
    result = scaler.execute()
    return {"scaled": result is not None}

API Reference

All mutation endpoints require an Idempotency-Key header. Responses use standard HTTP status codes.

Deployments

GET /api/v1/deployments List deployment actors

Query parameters: environment, namespace, limit (1–500), pageToken

GET /api/v1/deployments/summary Dashboard card summaries

Returns an array of deployment card summaries for the operations console.

PUT /api/v1/deployments/{env}/{ns}/{name} Register deployment actor

Body: {"owner":"...","serviceAccount":"flink","nodePool":"default"}

Starts a durable deployment actor workflow. Idempotent — safe to call repeatedly.

GET /api/v1/deployments/{env}/{ns}/{name}/actor Query actor state

Returns current version, health summary, active operation, recent operations, and autoscaler state.

GET /api/v1/deployments/{env}/{ns}/{name}/versions Version history

Returns the full version history (up to 50 most recent).

POST /api/v1/deployments/{env}/{ns}/{name}/deploy Submit controlled rollout

Body: {"requester":"...","approved":bool,"incident":bool,"reason":"...","spec":{...}}

Triggers a RolloutWorkflow: savepoint → apply → health gates → promote or rollback.

POST /api/v1/deployments/{env}/{ns}/{name}/scale Scale parallelism

Body: {"requester":"...","parallelism":16,"approved":bool,"reason":"..."}

POST /api/v1/deployments/{env}/{ns}/{name}/savepoint Request savepoint

Body: {"requester":"..."}

POST /api/v1/deployments/{env}/{ns}/{name}/suspend Suspend deployment

Takes a savepoint and stops the job. Resume to restart.

POST /api/v1/deployments/{env}/{ns}/{name}/resume Resume deployment

Restarts a suspended deployment and observes health.

POST /api/v1/deployments/{env}/{ns}/{name}/rollback Rollback to version

Body: {"requester":"...","targetVersion":3,"approved":true,"reason":"..."}

Omit targetVersion to roll back to the last healthy version.

Cluster Operations

GET /api/v1/clusters/{env}/{ns}/actor Cluster actor state

Returns freeze status and metadata for the namespace cluster actor.

POST /api/v1/clusters/{env}/{ns}/freeze Freeze namespace

Blocks all runtime mutations. Savepoints remain allowed.

POST /api/v1/clusters/{env}/{ns}/unfreeze Unfreeze namespace

Removes the freeze and allows mutations again.

Architecture

Temporal owns durable control-plane state. The Flink Kubernetes Operator remains the runtime reconciler. Flink handles data-plane execution and state. FCP orchestrates the lifecycle between them.

Workflow IDs

WorkflowID PatternPurpose
ClusterActorflink-cluster/{env}/{ns}Namespace-level freeze/unfreeze
DeploymentActorflink-deployment/{env}/{ns}/{name}Long-running per-job actor
RolloutWorkflowflink-rollout/{env}/{ns}/{name}/{opId}Controlled deploy/scale/rollback
SavepointWorkflowflink-savepoint/{env}/{ns}/{name}/{opId}Trigger + observe savepoint

Safety Model

  • Every deployment command requires an Idempotency-Key header — duplicates are safely ignored
  • Production operations classified as risky require explicit approval
  • Max-parallelism decreases are rejected after state exists unless a fresh start is approved
  • Job argument, operator UID, and job graph changes are treated conservatively
  • Stateful changes create a savepoint before applying
  • Resource-increasing changes acquire a time-bound capacity lease
  • Rollouts succeed only after runtime, checkpoint, restart, backpressure, lag, and sink health gates pass
  • Failed health gates automatically restore the previous healthy version
  • Savepoints remain allowed while a cluster is frozen; runtime mutations are blocked
